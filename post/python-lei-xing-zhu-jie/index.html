python-类型注解
2021-09-04

python 是动态语言，变量随时可以被赋值，且能赋值为不同类型，python 不是静态编译型语言，变量类型是在运行器决定的，动态语言很灵活，但是这种特性也是弊端，由于不做任何类型检查，直到运行期问题才显现出来，或者线上运行时才能暴露出问题。函数的使用者看到函数的时候，并不知道你的函数的设计，并不知道应该传入什么类型数据。IDE 也难以提示。

在此之前先介绍一下 函数注释

函数注释

在函数中对该函数 功能、参数 的解释。

def add(a,b):
    """
    param a: int 加数
    param b: int 加数
    return int 两数的和
    """
    return a+b

这个可以使用 help() 内置函数，获取其内容。一般的IDE都能对其给出提示。

但这玩意只能解决 函数的注释问题，对变量等并没有解决。而且这只是约定，并不是每个程序员都会写注释

为了解决这一痛点，python 引入类型注解这一概念

类型注解就是在 变量 函数 定义是为其标注上其类型。

语法：var: type = value

例：

name: str = "张三"

这样在IDE中就能给出更加精确的提示：

再结合 函数注释 给函数最精确的描述

这样 函数注释 只用描述函数的功能，不用再花大量篇幅解释 参数 返回值类型。

更何况这不止对函数有用。

typing

对于 list 等类型，类型注解是不知道里面的元素是什么类型的，只知道 变量 是一个列表 list 等类型。

names: list = ["张三", "李四"]
    # 类型注解是不知道names里面的元素是str类型的

这里就引入 typing 模块，它提供了很 ‘强’ 的类型支持。

对于上面的例子，就可以改成：

from typing import List
names: List[str] = ["张三", "李四"]

这是一些常用的类型注解：

from typing import List, Tuple, Set, Dict


a: List[int] = [1, 2, 3]

b: Tuple[int, int] = (1,2)

c: Set[str] = {"a", "b"}

d: Dict[str,str] = {"a":"A", "B":"b"}

写在最后

类型注解，并不会影响代码的运行，即你注解是int类型但还是可以改为其他类型。并不是强制性的，毕竟这是注解嘛。

养成良好的编码习惯是很重要的。
